# Week8 Unified Method and Baseline Plan - 2026-05-04

This plan extends the recovered pre-Cursor mainline after the shadow v1-v6
status audit. It defines the next evidence gates for deciding whether
`shadow_v6` becomes a paper-facing method or remains a bridge/ablation, and it
separates our self-trained framework evidence from external baseline evidence.

## Current Answer

We do have a self-trained model/framework line:

- `SRPD` is the current trainable ranking framework line.
- Week7.7 server export contains Beauty full973 and Books/Electronics/Movies
  small500 SRPD results.
- SRPD beats direct/structured-risk references on Beauty, Books, and
  Electronics in the checked week7.7 export.
- Movies remains the exception: structured-risk rerank is still stronger than
  SRPD in the checked week7.7 export.

We do not yet have a completed claim that our framework beats all external
baselines from the senior-recommended papers.

What currently exists for external baselines:

  `candidate_order_rank`, `popularity_prior_rank`, `longtail_prior_rank`,
  `history_overlap_rank`, `history_overlap_pairwise`, and
  `popularity_prior_pairwise`.
- A baseline protocol that correctly warns that reported numbers from other
  papers must not enter the same main table unless the protocol is matched.
- PDF collections under `Paper/BASELINE/NH` and `Paper/BASELINE/NR`, including
  LLM-rec, long-tail sequential recommendation, controllable recommendation,

What is still missing:

- Same-split, same-candidate, same-metric reproductions of strong external
  recommender baselines such as SASRec/BERT4Rec/GRU4Rec/LightGCN or the exact
  runnable methods selected from the senior-recommended papers.
- A paper-ready matrix where external baselines, direct LLM rankers,
  structured-risk, SRPD, shadow_v1, and shadow_v6 all have explicit protocol
  labels.
- Statistical tests and confidence intervals for any winner wording.

## Baseline Program

Baseline work should proceed in four layers.

### Layer 1: Already Runnable Task-Aligned Baselines

Use these as smoke/protocol baselines, not final external baselines:

- Candidate order.
- Popularity prior.
- Long-tail prior.
- History/title overlap.
- Pairwise history overlap.
- Pairwise popularity prior.

Purpose:

- Verify same-candidate metrics and schema.
- Catch leakage or candidate construction errors.
- Provide simple, interpretable lower bounds.

### Layer 2: Classical / RecBole-Style Recommender Baselines

Add or recover same-split baselines:

- SASRec.
- BERT4Rec.
- GRU4Rec.
- LightGCN or another graph collaborative-filtering baseline if data supports
  it.

Required adaptation:

- Train on train interactions only.
- Score the exact candidate set used by our ranking task.
- Evaluate with the same HR@10, NDCG@10, MRR, coverage, head exposure, and
  long-tail coverage fields.
- Do not compare full-catalog reported numbers against sampled-candidate
  results.

### Layer 3: Senior-Recommended Paper Baselines

Audit `Paper/BASELINE/NH` and `Paper/BASELINE/NR` into a runnable shortlist.

For each paper/project, record:

- Method name.
- Code availability.
- Dataset requirements.
- Whether it supports candidate-set reranking.
- Whether it supports sequential user histories.
- Whether it outputs calibrated confidence or only scores/ranks.
- Protocol gap if not runnable under our schema.

Initial categories visible from the PDF collection:

- LLM-enhanced sequential recommendation.
- Long-tail sequential recommendation.
- Controllable recommendation with LLM alignment.
- Unified multimodal / multi-task recommendation.
- Recent ACL/EMNLP/RecSys recommendation baselines.

Named local examples already visible from filenames:

- `SLMRec: Distilling Large...`
- `LLM-ESR: Large Language Models Enhancement for Long-Tailed Sequential Recommendation`
- `Aligning Large Language Models for Controllable Recommendations`
- `Towards Unified Multi-Mod...`
- ACL/EMNLP/ACM paper-id PDFs such as `2024.emnlp-main.*`,
  `2025.findings-acl.*`, `3626772.*`, `3696410.*`, `3711896.*`, and
  `3726302.*`.

These still need a proper paper-audit pass. Filename/title presence is not
enough to claim a runnable baseline.

Only promote a paper baseline when it can produce same-schema predictions or a
clearly labeled proxy table.

### Layer 4: Our Trainable Framework Line

Keep our trainable line separated from diagnostics:

- `SRPD`: current trainable ranking framework evidence.
- `shadow_v1 Signal LoRA`: future signal-learning stage if needed.
- `shadow_v6 Decision Bridge`: current diagnostic bridge.
- `Decision LoRA`: future stage only if shadow_v6 remains strong under aligned
  protocol.

SRPD is currently the only trainable line with four-domain server-exported
ranking results. Shadow LoRA is a design path, not a completed result.

### Week8.2: Classical Same-Candidate Baselines

Goal:

- Add or recover SASRec/BERT4Rec/GRU4Rec/LightGCN-style baselines.
- Convert their scores to the exact candidate-ranking evaluation schema.

Exit criteria:

- Four-domain candidate-set predictions.
- Same metrics as SRPD/shadow.
- Baseline rows enter unified matrix with `same_schema_external_baseline`.

First implementation order:

1. SASRec.
2. GRU4Rec.
3. BERT4Rec.
4. LightGCN, if the interaction graph is dense enough to justify it.

Adapter status:

- `main_export_same_candidate_baseline_task.py` exports train interactions,
  exact candidate rows, RecBole-style `.inter`, and metadata.
- `main_train_sasrec_same_candidate.py` trains a lightweight PyTorch SASRec
  baseline and writes exact-candidate scores without requiring RecBole.
- `main_train_gru4rec_same_candidate.py` trains a lightweight PyTorch GRU4Rec
  baseline through the same export/import protocol.
- `main_import_same_candidate_baseline_scores.py` imports
  `source_event_id,user_id,item_id,score` files and evaluates them under the
  existing ranking schema.
- No SASRec result is claimed until `sasrec_scores.csv` is imported with
  `status_label=same_schema_external_baseline` and full score coverage.
- SASRec is verified on the four-domain server matrix as the first
  `same_schema_external_baseline`.
- BERT4Rec/LightGCN training remains future work after SASRec/GRU4Rec are
  verified.
