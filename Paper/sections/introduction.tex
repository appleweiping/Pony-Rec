\section{Introduction}
\label{sec:introduction}

Large language models (LLMs) have become a common component in modern
recommendation systems. Recent methods use LLM-derived embeddings,
sequential adapters, graph representations, profile modules, or intent
representations to improve candidate scoring~\cite{llm2rec,llmemb,llmesr,irllrec,proex,promax,elmrec,rlmrec}.
These systems are normally judged by ranking metrics, but the decision problem
faced by a recommender is richer than rank alone: when should a high-scoring
candidate be trusted, and when should the system recognize that the evidence is
thin, conflicting, or near a decision boundary?

This paper studies actionable uncertainty for LLM-based recommendation under a
strict controlled evaluation setting. We focus on the reranking stage: each
method receives the same user histories and the same 101 candidates per event,
and all outputs are converted into the same score schema before metrics are
computed. This same-candidate protocol intentionally avoids claims about
full-catalog retrieval. It asks a narrower but sharper question: can
task-grounded uncertainty improve the reliability of candidate ranking when
the candidate set and evaluation schema are held fixed?

We propose \methodfull{} (\method), which turns LLM-derived relevance evidence
into a calibrated candidate posterior and an event-level uncertainty signal.
The uncertainty combines boundary ambiguity, calibration gap, evidence support,
and counterevidence. \method{} defines a validation-controlled ranking family:
one setting can downweight candidates whose relevance estimate is high but
poorly supported, while the $\eta=0$ setting reduces to the calibrated
posterior. Hyperparameters are fixed or selected on validation data only; test
rows are used only for final reporting and paired significance tests.

The resulting evidence supports a scoped conclusion. On four Amazon domains
(Sports, Toys, Home, and Tools), \method{} ranks first against eight
official-code-level baselines under the unified Qwen3-8B, same-candidate
protocol. The same evidence also limits the story: component ablations show
that several uncertainty terms are neutral, weak, or redundant, and
hyperparameter stability analyses support robustness rather than a claim that
every risk term is necessary. We therefore frame \method{} as an actionable
uncertainty framework for controlled reranking reliability, not as a universal
recommendation architecture.

Our contributions are:

\begin{enumerate}
    \item \textbf{Controlled official comparison.} We build a four-domain,
    10,000-user, 101-candidate same-candidate benchmark with eight
    official-code-level adapted LLM-based recommendation baselines,
    unified Qwen3-8B usage, exact score-coverage checks, full HR/NDCG/MRR
    reporting, paired Holm-corrected tests, and visible provenance
    summarized in Table~\ref{tab:baseline_provenance}.

    \item \textbf{Method.} We introduce \method, a calibrated candidate
    relevance posterior with task-grounded uncertainty components and a
    validation-controlled ranking family whose risk term is explicitly
    ablated.

    \item \textbf{Evidence discipline.} We pair the main comparison with
    paper-critical diagnostic modules: an uncertainty motivation study,
    leave-one-component-out ablations, hyperparameter stability analyses, and a
    framework overview. These modules define what can be claimed and what must
    be downgraded.
\end{enumerate}
