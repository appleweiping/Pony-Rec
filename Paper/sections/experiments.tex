\subsection{Experimental Setup}

\paragraph{Datasets.} We evaluate on four Amazon product-review domains:
Sports, Toys, Home, and Tools. Each domain uses 10,000 test users and a
101-candidate same-candidate task.

Each test instance consists of one ground-truth next item and 100 negatives.
The same candidate rows are shared by all methods. A result is table-eligible
only if its score file exactly matches the candidate key set, has no duplicate
or missing keys, contains finite numeric scores, and passes the local/server
provenance checks.

\paragraph{Backbone Model.} All methods use Qwen3-8B whenever an LLM or
LLM-derived representation is required. Baseline-specific adapters,
checkpoints, graph modules, intent modules, or representation learners are
preserved when they are part of the official algorithm.

\paragraph{Baselines.} We compare against eight official-code-level
LLM-based recommendation baselines:

\begin{itemize}
    \item \textbf{ELMRec}~\cite{elmrec}: Graph-enhanced LLM recommendation
    \item \textbf{IRLLRec}~\cite{irllrec}: Intent-aware retrieval with LLM
    \item \textbf{LLM2Rec}~\cite{llm2rec}: LLM-to-recommendation with SASRec
    \item \textbf{LLMEmb}~\cite{llmemb}: LLM embedding for recommendation
    \item \textbf{LLM-ESR}~\cite{llmesr}: LLM-enhanced sequential recommendation
    \item \textbf{ProEx}~\cite{proex}: Profile-based explanation recommendation
    \item \textbf{ProMax}~\cite{promax}: Profile-maximized recommendation
    \item \textbf{RLMRec}~\cite{rlmrec}: Reinforcement learning meets LLM recommendation
\end{itemize}

Baselines use official default or recommended hyperparameters unless a
documented bridge is required by the unified protocol. Test-set model selection
is forbidden for both baselines and \method.

\paragraph{Metrics and Statistics.} We report HR@$k$ and NDCG@$k$ for
$k\in\{5,10,20\}$, plus MRR. For each domain, \method{} is compared against
each official baseline on all seven metrics, yielding 56 paired tests
(8 baselines $\times$ 7 metrics). We use paired bootstrap confidence intervals
and Holm correction.

\paragraph{Evidence Packaging.} Every completed row has a lightweight local
evidence package with metrics, row counts, provenance, score audits, logs, and
server manifest hashes. Large score files, prediction files, checkpoints, and
embeddings remain server-side unless an archive decision requires otherwise.
